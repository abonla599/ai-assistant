# 工具接口说明

## 1. 工具系统概述
本系统基于装饰器注册机制，所有工具函数通过 `@register_tool` 声明后自动纳入全局工具表，供 AI 模型通过 Function Calling 调用。

## 2. 如何新增一个工具
1. 在 `backend/app/tools/builtin_tools.py`（或新建文件）中编写工具函数。
2. 使用 `@register_tool(name, description, parameters)` 装饰器注册，参数格式必须符合 OpenAI 工具定义的 JSON Schema。
3. 确保该文件在应用启动时被导入（例如在 `main.py` 或 `pipeline.py` 中导入）。
4. 工具函数接收关键字参数，返回字符串（推荐使用 `ToolResponse` 统一格式，成功以 `✓` 开头，失败以 `✗` 开头）。

示例（新增一个获取当前时间的工具）：
\`\`\`python
@register_tool(
    name="get_current_time",
    description="获取当前系统时间",
    parameters={"type": "object", "properties": {}, "required": []}
)
def get_current_time() -> str:
    from datetime import datetime
    return f"✓ 当前时间：{datetime.now()}"
\`\`\`

## 3. 现有工具列表

### 3.1 calculator（计算器）
- **功能**：执行数学表达式计算，支持 math 模块函数（sqrt, sin 等）。
- **参数**：
  - `expression` (string, 必填)：数学表达式，如 `"2+3*4"`。
- **返回**：成功返回 `✓ <结果>`，失败返回 `✗ 错误: ...`
- **限制**：禁止使用 `__`、`import` 等危险操作。

### 3.2 web_search（网络搜索）
- **功能**：使用 DuckDuckGo 搜索引擎获取实时信息。
- **参数**：
  - `query` (string, 必填)：搜索关键词。
- **返回**：成功返回 `✓ <搜索结果摘要列表>`，失败返回错误信息。
- **注意**：依赖网络，可能偶发超时。

### 3.3 help（帮助）
- **功能**：列出所有已注册工具及其描述。
- **参数**：无
- **返回**：字符串，每行一个工具名和简介。

### 3.4 execute_code（代码执行）
- **功能**：在安全沙箱中执行 Python 或 JavaScript 代码。
- **参数**：
  - `code` (string, 必填)：要执行的代码文本。
  - `language` (string, 可选，默认 `"python"`)：编程语言，可选 `"python"` 或 `"javascript"`。
- **返回**：格式 `输出:\n<stdout>\n错误:\n<stderr>`；若沙箱错误则返回 `执行错误: ...`
- **限制**：代码运行于隔离容器，无网络、内存 64MB、超时 10 秒，不允许提升权限。
