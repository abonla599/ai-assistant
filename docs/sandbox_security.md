# 沙箱安全审计报告

**编写人**：王翔宇  
**日期**：2026-05-10  
**版本**：v1.0

## 一、已采用的安全隔离措施

| 措施 | 说明 | 状态 |
|------|------|------|
| 非 root 用户执行 | 所有代码都以 sandbox 普通用户运行 | 已实施 |
| 只读文件系统 | 根目录 / 设为只读 | 已实施 |
| 临时可写目录 | 仅 /home/sandbox/tmp 可写，限制 32MB | 已实施 |
| 网络禁用 | 容器内无网络连接 | 已实施 |
| 禁止提权 | no-new-privileges:true | 已实施 |
| 剥夺内核能力 | --cap-drop=ALL | 已实施 |
| 内存限制 | 每个容器最多 64MB | 已实施 |
| CPU 限制 | 限制为 0.5 个核心 | 已实施 |
| 执行超时 | 默认 10 秒，超时即杀 | 已实施 |
| 代码只读挂载 | 用户代码以只读方式挂载 | 已实施 |
| 每日调用限制 | 每天最多 1000 次 | 已实施 |
| 自动清理 | 容器执行完立刻删除 | 已实施 |

## 二、风险点与缓解

| 风险 | 缓解措施 |
|------|----------|
| Docker 版本漏洞 | 定期更新 Docker Desktop |
| 拒绝服务攻击 | 每天 1000 次上限 |
| 内存超限 | Docker OOM 自动杀死 |
| 文件系统逃逸 | 只读挂载 + 禁用特权模式 |
| 内核漏洞提权 | no-new-privileges + cap-drop=ALL |

## 三、渗透测试结果

| 测试项 | 结果 |
|--------|------|
| 读取系统文件 | 被拒 |
| 写入系统目录 | 被拒（只读文件系统） |
| 联网请求 | 被拒（网络禁用） |
| 尝试提权 | 被拒 |
| 死循环 | 超时杀死 |
| 申请超大内存 | OOM 杀死 |
